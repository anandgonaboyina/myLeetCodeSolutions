<h2><a href="https://takeuforward.org/plus/">Find peak element</a></h2><h3>Medium</h3><hr><p>Given an array arr of integers. A peak element is defined as an element greater than both of its neighbors.</p><p>Formally, if arr[i] is the peak element, <strong>arr[i - 1] &lt; arr[i]</strong> and <strong>arr[i + 1] &lt; arr[i]</strong>.</p><p><br></p><p>Find the index(0-based) of a <strong>peak</strong> <strong>element</strong> in the array. If there are multiple peak numbers, return the index of any peak number.</p><p><br></p><p><strong>Note</strong>:</p><ul><li>As there can be many peak values, "<strong>true" </strong>is given as output if the returned index is a peak number, otherwise the returned value of index.</li></ul><br><br><p class="tuf-text-14">Example 1</p><div class="tuf-example text-new_secondary"><p><strong>Input </strong>: arr = [1, 2, 3, 4, 5, 6, 7, 8, 5, 1]</p><p><strong>Output</strong>: 7</p><p><strong>Explanation</strong>: In this example, there is only 1 peak that is at index 7.</p></div><br><br><p class="tuf-text-14 ">Example 2</p><div class="tuf-example text-new_secondary"><p><strong>Input </strong>: arr = [1, 2, 1, 3, 5, 6, 4]</p><p><strong>Output</strong>: 1</p><p><strong>Explanation</strong>: In this example, there are 2 peak numbers at indices 1 and 5. We can consider any of them.</p></div><br><br><p class="tuf-text-14 ">Constraints</p><div class="tuf-dark-content-box"><ul><li>&nbsp;1 &lt;= arr.length &lt;= 1000</li><li>&nbsp;-2<sup>31</sup> &lt;= arr[i] &lt;= 2<sup>31</sup> - 1</li><li>&nbsp;arr[i] != arr[i + 1] for all valid i.</li><li>For arr[0], its left element can be considered as -∞</li><li>For arr[n-1], its right element can be considered as -∞</li></ul></div><br><br>
